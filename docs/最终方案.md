既然全部标签已经完成，最终模型就按一次性收尾方案做，不再继续零散调参。

## 一、最终模型结构

保留已经验证有效的“固定四位置分类”路线，不改成 CTC、OCR Transformer 或字符切割模型。

最终结构建议使用约 **10 万参数的深度可分离 Position CNN**：

[
[B,1,32,96]\longrightarrow[B,4,10]
]

具体结构：

1. Stem
   `Conv 3×3, 1→32`，保持 (32\times96)。

2. Stage 1
   两个深度可分离残差块，通道 32。

3. Downsample 1
   `Conv 3×3, 32→48, stride=2`，得到 (16\times48)。

4. Stage 2
   两个深度可分离残差块，通道 48。

5. Downsample 2
   `Conv 3×3, 48→72, stride=2`，得到 (8\times24)。

6. Stage 3
   三个深度可分离残差块，通道 72，其中最后一个使用横向膨胀卷积：

[
\operatorname{dilation}=(1,2)
]

用于扩大字符间横向感受野。

7. 位置读取
   使用：

[
\operatorname{AdaptiveAvgPool2d}(1,4)
]

得到四个位置特征：

[
[B,72,1,4]\rightarrow[B,4,72]
]

8. 四个独立分类头
   每个位置分别使用：

[
72\rightarrow64\rightarrow10
]

不要共享最终分类头。第二、三位置明显更难，独立头能允许各位置形成不同决策边界。

建议使用：

* BatchNorm
* SiLU
* Dropout (0.1)
* 残差连接

预计单模型约 (9\万)–(12\万) 参数，三个种子集成后仍低于 40 万参数，ONNX 大约 1–2 MB。

## 二、数据切分

不要立即把所有图片直接投入训练，否则最终模型仍然没有可信准确率。

从全部标注数据中冻结一个最终测试集：

[
N_{\mathrm{test}}=\max(300,;15%N)
]

测试集应覆盖实际部署中的全部验证码来源，并在各来源内近似平衡四个位置的数字边际分布。当前数据量下按来源近似等额抽取，而不是只冻结某一个来源的最新批次。测试集需要满足：

* 不参与增强参数选择；
* 不参与 epoch 选择；
* 不参与模型结构选择；
* 只在最终模型确定后运行一次。

剩余数据划分为：

[
90%\text{训练}+10%\text{验证}
]

验证集按数据源、批次和数字类别进行分层。

最终会形成两个版本：

* `captcha-final-eval`：不使用测试集训练，用于得到可信指标；
* `captcha-final-prod`：完成测试后，将测试集也合并进去重新训练，用于实际部署。

生产模型可以使用全部数据，但报告里的准确率必须来自 `eval` 模型。

## 三、损失函数

继续使用四位置交叉熵：

[
L=\sum_{i=1}^{4}w_i\operatorname{CE}(z_i,y_i)
]

位置权重固定为：

[
(w_1,w_2,w_3,w_4)=(1.0,1.15,1.15,1.0)
]

因为此前第二、三位置准确率明显低于第一、四位置。

使用轻度标签平滑：

[
\varepsilon=0.02
]

不要使用 focal loss，也不要针对 `1↔7` 等混淆对手工修改类别权重。混淆来自字形相似，而不是简单类别不平衡。

## 四、数据增强

增强必须保持验证码字符语义，不要使用过强形变。

建议固定为：

| 增强   |              范围 |
| ---- | --------------: |
| 水平平移 |       (\pm2) px |
| 垂直平移 |       (\pm1) px |
| 旋转   |    (\pm2^\circ) |
| 缩放   |  (0.95\sim1.05) |
| 对比度  |    (0.9\sim1.1) |
| 亮度   |    (0.9\sim1.1) |
| 高斯噪声 | (\sigma\le0.03) |
| 高斯模糊 |    小概率，kernel 3 |

禁止：

* 水平翻转；
* 大角度旋转；
* 大幅随机裁剪；
* MixUp；
* CutMix；
* 强弹性形变；
* 单字符局部擦除。

这些增强容易把一个数字变成另一个数字，尤其会加剧 `1/7`、`6/8`、`2/3` 的混淆。

## 五、训练配置

使用固定更新次数，而不是固定 epoch。

推荐配置：

| 项目                | 设置                 |
| ----------------- | ------------------ |
| Optimizer         | AdamW              |
| 初始学习率             | (1.5\times10^{-3}) |
| Weight decay      | (1\times10^{-4})   |
| Batch size        | 32 或 64            |
| Warmup            | 500 updates        |
| Scheduler         | cosine decay       |
| 最低学习率             | (1\times10^{-5})   |
| 总更新次数             | 25,000             |
| Gradient clipping | 1.0                |
| EMA decay         | 0.999              |
| Seeds             | 42、3407、20260716   |

每个种子都保存：

* 原始最佳权重；
* EMA 最佳权重。

最终优先使用 EMA 权重。模型选择指标只看：

[
\operatorname{ExactAcc@1}
]

CharAcc 只能作为辅助指标。

验证集连续 4000 次更新没有刷新 ExactAcc 时，可以提前终止，但最低至少训练 15,000 次更新。

## 六、困难样本回放

在前 70% 更新中正常均匀采样。

训练进行到约 17,500 次更新后，对训练集完整推理一次，找出：

* 整码识别错误；
* 第二、三位置识别错误；
* 低置信度正确样本。

最后 7500 次更新中，将这些困难样本的采样概率提高到普通样本的 2 倍，但最高不要超过 3 倍。

这比重新设计网络或使用 focal loss更稳定。

## 七、最终集成

训练三个完全相同的模型：

[
M_{42},\quad M_{3407},\quad M_{20260716}
]

对 logits 求平均：

[
z_{\mathrm{final}}
==================

\frac{
z_{42}+z_{3407}+z_{20260716}
}{3}
]

再分别对四个位置取 argmax：

[
\hat y_i=\arg\max_c z_{\mathrm{final},i,c}
]

把三个模型直接封装进一个 ONNX，接口继续保持：

[
\texttt{input}\ [B,1,32,96]
\rightarrow
\texttt{logits}\ [B,4,10]
]

部署端不需要修改现有接口。

## 八、最终验收标准

最终测试集至少报告：

* CharAcc；
* ExactAcc@1；
* ExactAcc@2/3/5；
* 四位置准确率；
* 每个数字的 precision、recall；
* 混淆矩阵；
* 三个单模型结果；
* 集成结果；
* PyTorch/ONNX logits 最大误差。

最低发布线设为：

[
\boxed{\operatorname{ExactAcc@1}\ge60%}
]

理想目标：

[
\boxed{\operatorname{ExactAcc@1}\ge65%}
]

对应每次失败刷新新验证码、最多提交三次时：

[
p=60%
\Rightarrow
1-(1-p)^3=93.6%
]

[
p=65%
\Rightarrow
1-(1-p)^3\approx95.71%
]

即使最终仍只有 (55%)：

[
1-0.45^3\approx90.89%
]

也已经具有较强实用性。

## 最终执行顺序

[
\boxed{
\begin{aligned}
&\text{按不同来源和数字边际分层冻结测试集}\
\rightarrow;&\text{训练 Position-DS CNN 三个种子}\
\rightarrow;&\text{使用 EMA 权重}\
\rightarrow;&\text{最后 30% 更新做困难样本回放}\
\rightarrow;&\text{三模型 logits 集成}\
\rightarrow;&\text{测试集只评估一次}\
\rightarrow;&\text{合并全部数据训练生产版}\
\rightarrow;&\text{导出单一 ONNX}
\end{aligned}
}
]

这就是最终模型方案。核心变化不是继续堆参数，而是从原来的 40.4k 普通宽版 CNN，升级为约 10 万参数、保留横向空间结构、四位置独立头的深度可分离残差 CNN。

## 实现落地说明（2026-07-16）

全部 3203 张样本完成标注后，完整冻结单一来源的最新 1000 张会占总数据的 31.2%，而且无法衡量其他来源。根据测试集下限公式和后续确认，实际实现冻结 481 张，即 `max(300, ceil(15%N))`，并按来源近似等额分配：新优酷 161 张、次元城动画 160 张、饭团动漫 160 张；各来源内部再近似平衡四个位置的数字边际。默认最终切分为 2447 训练、275 验证、481 测试，生产版在测试完成后仍合并全部 3203 张重训。

历史实验 manifest 已覆盖 2477 张样本。为了兼顾来源覆盖，当前测试集中有 218 张曾参与前轮结构实验：次元城动画 159 张、饭团动漫 59 张、新优酷 0 张。这些样本不得进入最终 eval 模型训练，但由于曾影响方案选择，报告只能称为“多来源留出评估”，不能称为完全未见盲测。要恢复完全独立的多来源盲测，至少还需新增并人工标注 159 张次元城动画和 59 张饭团动漫样本。

正式训练必须支持断点续训。实现默认每 250 updates 原子保存模型、EMA、optimizer、update、最佳指标、困难样本权重、采样器生成器和 Python/NumPy/PyTorch 随机状态。恢复时必须复用原 split 和训练配置；已完成的 seed 直接跳过，未完成的 seed 从最近断点继续。只恢复模型权重而重新初始化 optimizer、EMA 或采样器不算有效续训。
