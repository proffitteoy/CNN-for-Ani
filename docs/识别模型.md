那只谈机器学习部分，我建议你把问题明确成：

[
\textbf{固定长度 4 位数字验证码的多头分类（multi-head classification）}
]

而不是 OCR，也不做字符检测、分割、CTC。

模型就是

[
f_\theta:\mathbb R^{1\times H\times W}\to \mathbb R^{4\times 10}
]

其中第 (i) 个 head 预测第 (i) 位数字：

[
p_i=\operatorname{softmax}(z_i),\qquad z_i\in\mathbb R^{10}
]

损失函数直接四个交叉熵相加：

[
\mathcal L
==========

\sum_{i=1}^{4}
\operatorname{CE}(z_i,y_i)
]

或者取平均，没有本质区别。

我建议第一版模型甚至这么写：

```python
class CaptchaCNN(nn.Module):
    def __init__(self):
        super().__init__()

        self.features = nn.Sequential(
            nn.Conv2d(1, 8, 3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2),

            nn.Conv2d(8, 16, 3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2),
        )

        self.classifier = nn.Sequential(
            nn.Flatten(),
            nn.Linear(16 * 8 * 24, 40),
        )

    def forward(self, x):
        x = self.features(x)
        x = self.classifier(x)

        return x.view(-1, 4, 10)
```

假设输入：

[
32\times96
]

那么 feature map 流程：

[
1\times32\times96
\to
8\times16\times48
\to
16\times8\times24
]

最后输出：

[
4\times10
]

但这里有一个明显问题：最后这个 Linear 参数量是

[
(16\cdot8\cdot24)\times40
=========================

122880
]

其实对你的场景仍然偏大。

我的建议是继续压：

```python
class CaptchaCNN(nn.Module):
    def __init__(self):
        super().__init__()

        self.features = nn.Sequential(
            nn.Conv2d(1, 8, 3, stride=2, padding=1),
            nn.ReLU(),

            nn.Conv2d(8, 16, 3, stride=2, padding=1),
            nn.ReLU(),

            nn.Conv2d(16, 16, 3, stride=2, padding=1),
            nn.ReLU(),
        )

        self.position_pool = nn.AdaptiveAvgPool2d((4, 4))
        self.classifier = nn.Linear(16 * 4, 10)

    def forward(self, x):
        x = self.features(x)
        x = self.position_pool(x)
        x = x.permute(0, 3, 1, 2).flatten(2)

        return self.classifier(x)
```

这里保留 4 行垂直特征，只把 12 列横向特征聚合成 4 个固定位置。四个位置共享同一个数字分类器，输出仍为 `[B, 4, 10]`，总参数量为 4218。

直接池化为 `(1, 4)` 会丢失过多垂直笔画信息；真实 32 样本过拟合检查表明 `(4, 4)` 更适合当前验证码。

这才比较符合你“我要自己在 Kotlin 写前向传播”的目标。

核心训练逻辑极其简单：

```python
logits = model(images)  # [B, 4, 10]

loss = sum(
    F.cross_entropy(logits[:, i, :], labels[:, i])
    for i in range(4)
)

optimizer.zero_grad()
loss.backward()
optimizer.step()
```

准确率不要只看字符准确率。

你真正关心的是验证码整体准确率：

[
\operatorname{Acc}_{captcha}
============================

P(
\hat y_1=y_1,
\hat y_2=y_2,
\hat y_3=y_3,
\hat y_4=y_4
)
]

代码：

```python
pred = logits.argmax(dim=-1)

captcha_acc = (
    pred.eq(labels)
    .all(dim=1)
    .float()
    .mean()
)
```

原因很简单。假设单字符准确率是 (98%)，近似独立时：

[
0.98^4\approx92.2%
]

对于自动验证码填写，这其实不算特别高。

你至少应该追求：

[
\text{4-digit exact accuracy}>98%
]

最好 (99%) 左右。

推荐：

[
\text{translation}
+
\text{brightness}
+
\text{contrast}
+
\text{small rotation}

例如：

```python
transforms.Compose([

    transforms.ToTensor(),

```

我明确不建议：

```text
horizontal flip
vertical flip
large rotation
random crop
```

因为数字结构不是这些变换下不变的。

尤其：

[
6\leftrightarrow9
]

旋转增强太大会直接污染标签语义。

---

还有一个关键设计：置信度。

不要单纯：

```python
argmax
```

模型输出四个位的概率：

[
c_i=\max_j p_{ij}
]

定义验证码置信度：

[
C=\min_{1\le i\le4}c_i
]

我建议用 min，而不是平均值。

例如：

```text
9: 0.999
2: 0.998
7: 0.531
1: 0.997
```

平均值仍然有：

[
0.881
]

但显然第三位非常可疑。

所以：

```python
confidence = probs.max(dim=-1).values.min(dim=-1).values
```

业务逻辑就可以变成：

```text
confidence >= 0.95
→ 自动填写并提交

confidence < 0.95
→ 不自动提交
```

这对验证码场景很重要，因为错误提交可能直接刷新验证码。
