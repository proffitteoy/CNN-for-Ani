可以。把整个训练流程压成四个阶段，机器学习部分可以这样设计。

## 第一阶段：定位并下载验证码图片

目标不是立即训练，而是先确认验证码的真实数据分布。

首先定位 Animeko 当前验证码出现的位置，追踪：

[
\text{验证码检测}
\rightarrow
\text{图片请求 URL}
\rightarrow
\text{HTTP Response}
\rightarrow
\text{Image ByteArray}
]

在验证码图片进入 UI 前，把原始图片旁路保存下来。

建议保存原图，不做 resize、灰度化等预处理：

```text
raw/
├── 000001.png
├── 000002.png
├── 000003.png
└── ...
```

同时保存最少的元数据：

```text
metadata.csv
```

例如：

```csv
id,source,width,height
000001,source_a,120,40
000002,source_a,120,40
000003,source_b,100,32
```

这一阶段主要判断四件事：

[
\begin{aligned}
&1.\ \text{是否始终为 4 位数字}\
&2.\ \text{图片尺寸是否固定}\
&3.\ \text{验证码生成器是否固定}\
&4.\ \text{字符是否存在粘连、旋转、干扰线}
\end{aligned}
]

建议先收集：

[
N\geq1000
]

如果能够自动批量请求验证码，则收集：

[
3000\sim10000
]

张。

这里最重要的是避免训练集只来自一次请求。验证码生成器内部可能有字体池、颜色池和背景模板，因此数据最好跨多个时间批次采集。

最终输出：

```text
dataset/
└── raw/
```

以及一份验证码分布分析结果。

## 第二阶段：样本分类、标注与 K 折划分

这一阶段解决：

[
I\mapsto(y_1,y_2,y_3,y_4)
]

也就是为每张验证码赋真实标签。

最简单的标注方式就是人工查看图片并输入四位数字：

```text
000001.png → 5271
000002.png → 0834
```

标注完成后统一重命名：

```text
labeled/
├── 5271_000001.png
├── 0834_000002.png
├── 9921_000003.png
└── ...
```

注意不能只使用：

```text
5271.png
```

因为相同验证码可能出现多次，会发生文件名冲突。

标签解析：

```python
label = filename.split("_")[0]

target = torch.tensor([
    int(label[0]),
    int(label[1]),
    int(label[2]),
    int(label[3]),
])
```

然后先做数据检查。

统计每个位置数字的分布：

[
P(Y_i=k),
\qquad
i=1,2,3,4,\quad k=0,\ldots,9
]

理想情况下每一位应该近似均匀：

[
P(Y_i=k)\approx0.1
]

例如生成：

```text
position 0:
0: 487
1: 503
...
9: 491
```

如果某些数字严重缺失，需要继续补数据。

### K 折划分

建议：

[
K=5
]

使用 5-fold cross validation。

整个数据集划分为：

[
D=D_1\cup D_2\cup\cdots\cup D_5
]

每轮：

[
\begin{array}{c|c}
\text{Fold}&\text{Validation}\
\hline
1&D_1\
2&D_2\
3&D_3\
4&D_4\
5&D_5
\end{array}
]

剩余四折作为训练集。

这里最好使用近似分层划分。

但是验证码是：

[
10^4=10000
]

类组合，直接对完整验证码字符串做 stratified split 没有意义。

可以构造一个近似 stratification key，例如：

```python
key = f"{label[0]}_{label[1]}"
```

或者使用 iterative stratification，使四个位上的数字边际分布尽量保持一致。

核心要求是：

[
P_{\text{train}}(Y_i=k)
\approx
P_{\text{val}}(Y_i=k)
]

另外，如果图片是按批次下载：

```text
batch_20260701
batch_20260702
batch_20260703
```

我反而建议使用 Group K-Fold：

[
\text{group}=\text{download batch}
]

避免同一个验证码生成状态下的高度相似样本同时进入训练集和验证集。

最终输出：

```text
dataset/
├── raw/
├── labeled/
└── folds.json
```

例如：

```json
{
  "000001": 0,
  "000002": 3,
  "000003": 1
}
```

## 第三阶段：模型训练

固定输入：

[
I\in\mathbb R^{1\times32\times96}
]

预处理：

[
\text{RGB}
\rightarrow
\text{grayscale}
\rightarrow
32\times96
\rightarrow
[0,1]
]

最好保持预处理极简单，因为之后需要在 Kotlin 中完全复现。

例如：

```python
image = image.convert("L")
image = image.resize((96, 32))

x = np.asarray(image, dtype=np.float32) / 255.0
```

不要训练时使用复杂的 ImageNet normalization。

模型：

[
1\times32\times96
\rightarrow
\operatorname{Conv}(1,8)
\rightarrow
\operatorname{ReLU}
\rightarrow
\operatorname{Conv}(8,16)
\rightarrow
\operatorname{ReLU}
\rightarrow
\operatorname{Conv}(16,16)
\rightarrow
\operatorname{ReLU}
\rightarrow
\operatorname{Linear}
\rightarrow
4\times10
]

损失：

[
\mathcal L
==========

\frac14
\sum_{i=1}^4
\operatorname{CE}(z_i,y_i)
]

训练：

```python
for images, labels in train_loader:
    logits = model(images)

    loss = sum(
        F.cross_entropy(
            logits[:, i],
            labels[:, i],
        )
        for i in range(4)
    ) / 4

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
```

第一版直接：

```text
Optimizer: AdamW
Learning rate: 1e-3
Batch size: 64
Epoch: 50
```

加：

```text
Early stopping
patience = 8
```

评价指标必须至少包含三个。

字符准确率：

[
\operatorname{CharAcc}
======================

\frac{
\sum_{n=1}^N\sum_{i=1}^4
\mathbf 1(\hat y_{ni}=y_{ni})
}{
4N
}
]

完整验证码准确率：

[
\operatorname{ExactAcc}
=======================

\frac1N
\sum_{n=1}^N
\mathbf1(
\hat{\mathbf y}_n=\mathbf y_n
)
]

错误验证码平均错误字符数：

[
\operatorname{HammingError}
===========================

\frac1N
\sum_{n=1}^N
\sum_{i=1}^4
\mathbf1(
\hat y_{ni}\neq y_{ni}
)
]

重点看：

[
\boxed{\operatorname{ExactAcc}}
]

5 折最终得到：

```text
Fold 1 Exact Acc: 99.12%
Fold 2 Exact Acc: 98.93%
Fold 3 Exact Acc: 99.05%
Fold 4 Exact Acc: 99.20%
Fold 5 Exact Acc: 98.88%
```

汇报：

[
\mu\pm\sigma
]

例如：

[
99.04%\pm0.13%
]

K 折的目的主要是判断模型是否稳定，不是最终输出五个模型。

模型结构确定之后，用全部训练数据：

[
D_1\cup\cdots\cup D_5
]

重新训练一个 final model。

这里可以根据 K 折结果选择 epoch 数，例如五折最佳 epoch：

```text
21, 25, 23, 27, 24
```

取：

[
\operatorname{median}=24
]

然后最终模型使用全部数据训练 24 epoch。

## 第四阶段：输出可使用模型

这里不建议最终直接输出 `.pt` 或 `.onnx`。

因为你的目标是纯 Kotlin 前向传播。

训练模型：

```text
captcha.pt
```

只是 Python 训练阶段产物。

然后写一个 exporter：

```text
export_model.py
```

提取：

```python
state_dict = model.state_dict()
```

包含：

```text
features.0.weight
features.0.bias
features.2.weight
features.2.bias
features.4.weight
features.4.bias
classifier.weight
classifier.bias
```

导出成自己的二进制格式：

```text
captcha.weights
```

结构例如：

```text
CAPTCHA_MODEL_V1

conv1.weight
shape: 8,1,3,3
float32 data

conv1.bias
shape: 8
float32 data

conv2.weight
shape: 16,8,3,3
float32 data

...

linear.weight
shape: 40,768

linear.bias
shape: 40
```

实际二进制不用保存字段名。

可以定义固定结构：

```text
Header
├── magic: 4 bytes
├── version: int32
└── dtype: uint8

Weights
├── conv1.weight
├── conv1.bias
├── conv2.weight
├── conv2.bias
├── conv3.weight
├── conv3.bias
├── linear.weight
└── linear.bias
```

Kotlin 端严格按照顺序读取。

例如：

```kotlin
data class CaptchaModel(
    val conv1Weight: FloatArray,
    val conv1Bias: FloatArray,
    val conv2Weight: FloatArray,
    val conv2Bias: FloatArray,
    val conv3Weight: FloatArray,
    val conv3Bias: FloatArray,
    val linearWeight: FloatArray,
    val linearBias: FloatArray,
)
```

最终需要做一次 Python/Kotlin 一致性测试。

给定同一张图片 (I)：

[
z_{\text{PyTorch}}=f_{\text{PyTorch}}(I)
]

[
z_{\text{Kotlin}}=f_{\text{Kotlin}}(I)
]

检查：

[
\max_i
|z_i^{\text{PyTorch}}
---------------------

z_i^{\text{Kotlin}}|
<
10^{-4}
]

不能只检查最终识别结果都是 `"5271"`。

必须检查 logits。

因为可能：

```text
PyTorch:  [0.1, 0.2, 8.1, ...]
Kotlin:   [0.1, 0.2, 5.4, ...]
```

argmax 相同，但前向传播实际上已经出现误差。

最终项目真正使用的产物只有：

```text
captcha.weights
```

完整流水线就是：

[
\boxed{
\begin{aligned}
&\text{定位验证码请求并采集原图}\
&\downarrow\
&\text{人工标注 + 数据分布检查 + 5 Fold}\
&\downarrow\
&\text{小型 CNN 交叉验证 + 全数据重训}\
&\downarrow\
&\text{PyTorch 权重导出为 captcha.weights}\
&\downarrow\
&\text{Kotlin 实现前向传播并进行 logits 对齐测试}
\end{aligned}
}
]

我的判断是：这套流程已经足够接近一个小型、规范的 ML 工程，而不是“随便训个 CNN 塞进去”。尤其是 **按下载批次做 Group K-Fold** 和 **PyTorch/Kotlin logits 对齐测试**，这两个地方最值得保留。
